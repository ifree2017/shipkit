# Contract Boundary Checklist

## Deliverables

- [ ] Every deliverable is clearly defined.
- [ ] Every deliverable has acceptance criteria.
- [ ] No “best effort” deliverables are hidden in scope.

## Timeline

- [ ] Milestones are defined.
- [ ] Client cooperation deadlines are defined.
- [ ] Delay responsibility is explicit.

## Change Control

- [ ] Change request process is defined.
- [ ] Out-of-scope definition is clear.
- [ ] Price/timeline impact process is defined.

## Risk Allocation

- [ ] Known risks are listed.
- [ ] Third-party dependency responsibility is assigned.
- [ ] Data/security responsibility is assigned.

## Acceptance

- [ ] Acceptance cycle is defined.
- [ ] Sign-off owner is named by role.
- [ ] Non-response rule is defined.

## Legal Review Items

- [ ] IP ownership
- [ ] Data security liability
- [ ] Confidentiality
- [ ] Exclusivity / non-compete
- [ ] Liability limitation
- [ ] Indemnification
- [ ] Penalty clauses

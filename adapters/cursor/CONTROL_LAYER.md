# Cursor Adapter: ShipKit Control Layer

Recommended mapping:

- Put high-level rules in `.cursor/rules/shipkit.mdc`.
- Keep detailed workflows under `workflows/` and `protocol/`.
- Keep skills as markdown references if native skill loading is not available.
- Use `sk check` in terminal before PR or delivery.

Cursor is best used as an IDE runner. ShipKit should keep the core facts in the repository so other platforms can continue from the same state.

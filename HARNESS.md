# ShipKit Harness

ShipKit defines a portable delivery harness for agent-based software projects.

## Stages

1. Intake
2. Discover
3. Scope
4. PRD
5. Architecture
6. Plan
7. Build
8. Test
9. Release
10. Delivery

## Principles

- Project facts live in files, not chat history.
- Every stage has inputs, outputs, and gates.
- Client-facing artifacts must not leak internal pricing, assumptions, or sensitive information.
- Agent outputs must be reviewable, auditable, and portable across platforms.

# Workflow: Trace

Use to validate requirement-to-delivery evidence.

## Steps

1. List requirements.
2. Link to modules, PRD sections, architecture components, tasks, tests, release, and delivery evidence.
3. Identify orphans.
4. Repair missing links or create issues.
5. Block delivery if critical trace is missing.

## Outputs

- trace/trace-map.md
- trace/requirement-map.md
- trace/test-map.md
- trace/release-map.md

# debt Workflow

Use the "debt" skill when this flow is triggered.

## Steps
1. Detect trigger.
2. Create the corresponding issue or artifact from templates when available.
3. Classify severity, stage impact, owner, and next action.
4. Update related scope, PRD, architecture, plan, test, release, or delivery documents when applicable.
5. Produce a handoff note if the flow changes stage readiness.

## Gate
Run the related gate in gates/ before closing the flow.

# Security

Security must be checked at PRD, architecture, and release stages.

For each security review, check:
1. Authentication
2. Authorization
3. Sensitive data
4. Log redaction
5. Token and secret storage
6. File upload risks
7. Admin operations
8. Audit logs
9. Backup and deletion
10. Third-party exposure

Security issues must include severity, affected asset, mitigation, and verification.

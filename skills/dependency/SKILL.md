# Dependency

External dependencies must be tracked before quote, plan, release, or delivery.

For each dependency, produce:
1. Dependency name
2. Responsible party
3. Required input or access
4. Needed-by stage
5. Affected modules
6. Impact if missing
7. Fallback or workaround
8. Quote/contract implication
9. Status

Classify as:
- P0 blocker
- P1 schedule risk
- P2 quality risk
- P3 optional

Customer-owned dependencies must be written as preconditions or open questions.

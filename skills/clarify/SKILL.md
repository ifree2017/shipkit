# Clarify

Do not convert unclear requirements into scope or tasks.

First classify the item:
- must-confirm: cannot continue without answer
- should-confirm: can proceed with risk
- nice-to-confirm: can proceed with explicit assumption

For each clarification, produce:
1. Question
2. Context
3. Why it matters
4. Impact if unanswered
5. Proposed default assumption, if safe
6. Owner
7. Due date
8. Recommended next step

If the answer changes scope, convert to change. If it reveals a defect against accepted scope, convert to bug.

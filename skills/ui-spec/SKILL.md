# UI Spec

Inputs:

- PRD
- Scope
- Acceptance criteria
- DESIGN.md when available

Outputs:

- `design/ui-spec.md`
- component list
- interaction states
- responsive behavior
- accessibility notes
- visual acceptance criteria

Every UI spec must include normal, empty, loading, error, permission-denied, and success states when relevant.

# status

Generate project status reports from artifacts.

Read:
- STATE.md
- modules/progress.md
- reports/progress-report.md
- reports/workload-report.md
- tasks/board.yaml
- blockers/
- risks/
- changes/
- defects/
- refactors/
- sync/daily-log.md

Produce or update:
- reports/status-report.md
- sync/daily-log.md

Rules:
- Status must include progress, blockers, risks, workload, changes/bugs/refactors, decisions needed, and next actions.
- Do not hide blockers or unresolved decisions.
- Separate internal status from client-facing status when needed.

Output:
1. Executive summary
2. Module progress
3. Workload status
4. Risks/blockers
5. Decisions needed
6. Next actions

# Scope Drift

Scope drift must be classified before it becomes work.

Watch for phrases:
- also support
- by default
- quick change
- simple addition
- while you're there
- should be easy
- just add
- expected behavior

Classify as:
1. Already in scope
2. Clarification
3. Change request
4. Bug
5. Future enhancement
6. Reject / out of scope

Never silently add scope drift to the build plan.
